# 06 — Roadmap

Este roadmap existe para manter o projeto pequeno, honesto e progressivo.

A prioridade não é publicar rápido.

A prioridade é documentar bem a jornada.

## Fase 1 — Base privada

Objetivo:

```txt
Organizar os primeiros documentos e registrar a visão inicial.
```

Status:

```txt
Em andamento
```

Entregas:

- [x] Criar repositório privado.
- [x] Criar README inicial.
- [x] Criar documento da história.
- [x] Criar princípios centrais.
- [x] Criar documento de transparência sobre IA.
- [x] Criar fluxo problema → produto.
- [x] Criar guia inicial de trabalho com IA.
- [x] Criar lições aprendidas.

## Fase 2 — Refinamento da narrativa

Objetivo:

```txt
Transformar os documentos iniciais em uma narrativa clara e acessível.
```

Possíveis tarefas:

- [ ] Revisar README para ficar mais humano.
- [ ] Melhorar `00_MY_STORY.md` com mais detalhes pessoais.
- [ ] Adicionar exemplos reais do Roteirizador.
- [ ] Adicionar exemplos reais do Casa em Dia.
- [ ] Separar o que é experiência pessoal do que é método geral.
- [ ] Definir tom: honesto, prático, acessível e sem promessa exagerada.

## Fase 3 — Método básico

Objetivo:

```txt
Transformar a jornada em um método simples para outras pessoas seguirem.
```

Possíveis documentos:

- Como encontrar problemas reais.
- Como descrever um problema para uma IA.
- Como definir um MVP.
- Como validar se a IA entregou certo.
- Como documentar sem burocracia.
- Como evitar alucinações.

## Fase 4 — Estudos de caso

Objetivo:

```txt
Mostrar exemplos reais em vez de apenas conceitos.
```

Casos possíveis:

- Roteirizador.
- Casa em Dia.
- AISDD.
- Pequenas automações pessoais.

Cada estudo de caso deve mostrar:

- problema;
- contexto;
- primeira solução;
- erros;
- evolução;
- papel da IA;
- aprendizados.

## Fase 5 — Publicação futura

Objetivo:

```txt
Avaliar se o projeto deve se tornar público.
```

Critérios antes de publicar:

- [ ] Conteúdo revisado.
- [ ] Uso de IA declarado claramente.
- [ ] Exemplos sensíveis removidos ou anonimizados.
- [ ] Nenhuma informação privada de empresa.
- [ ] Narrativa coerente.
- [ ] Promessas exageradas removidas.

## Princípio de evolução

```txt
Não transformar este projeto em produto antes de entender o que ele realmente é.
```

Primeiro documentar.

Depois organizar.

Depois decidir se deve ser publicado.
